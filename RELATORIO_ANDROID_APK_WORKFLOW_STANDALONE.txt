1. PROBLEMA IDENTIFICADO
================================================================================

Necessidade de Build APK Independente:

❌ ANTES:
   - APK gerado apenas durante pipeline completo (test + build + release)
   - Sem opção de build APK sob demanda
   - Sem flexibilidade para testes rápidos
   - Sem workflow separado para APK

NECESSIDADE:
   - Build APK independente
   - Executável manualmente via GitHub Actions
   - Sem necessidade de passar por test job
   - Mais rápido para desenvolvimento

3. DETALHES TÉCNICOS
================================================================================

3.1 TRIGGER (QUANDO EXECUTAR)
─────────────────────────────────────────────────────────────────────────────

```yaml
on:
  push:
    branches:
      - main
  workflow_dispatch:
```

Triggers:

1. Push para main
   └─ Executa automaticamente
   └─ Toda vez que há push em main
   └─ Gera APK automaticamente

2. workflow_dispatch
   └─ Executável manualmente
   └─ GitHub Actions → Build APK → Run workflow
   └─ Permite build sob demanda

Benefício:
- Automático: APK gerado em cada push
- Manual: Pode executar a qualquer momento
- Flexível: Não depende de pipeline completo

3.2 JOB CONFIGURATION
─────────────────────────────────────────────────────────────────────────────

```yaml
jobs:
  build-apk:
    runs-on: ubuntu-latest
```

Configuração:

- Job name: build-apk
- Runner: ubuntu-latest
- Imagem: Ubuntu 22.04 com ferramentas pré-instaladas
- Tempo: ~15-20 minutos

3.3 STEPS (PASSOS DO WORKFLOW)
─────────────────────────────────────────────────────────────────────────────

Step 1: Checkout Repository
```yaml
- name: Checkout Repository
  uses: actions/checkout@v3
```
└─ Faz checkout do código
└─ Tempo: ~10 segundos

Step 2: Setup Node
```yaml
- name: Setup Node
  uses: actions/setup-node@v3
  with:
    node-version: 20
```
└─ Instala Node.js v20
└─ Tempo: ~30 segundos

Step 3: Enable Corepack
```yaml
- run: corepack enable
```
└─ Ativa Corepack (gerenciador de package managers)
└─ Tempo: ~5 segundos

Step 4: Prepare pnpm
```yaml
- run: corepack prepare pnpm@latest --activate
```
└─ Instala/atualiza pnpm
└─ Tempo: ~30 segundos

Step 5: Install Dependencies
```yaml
- name: Install Dependencies
  run: pnpm install
```
└─ Instala dependências do projeto
└─ Tempo: ~2-3 minutos

Step 6: Build Web Assets
```yaml
- name: Build Web Assets
  run: pnpm build
```
└─ Compila código web (React + TypeScript)
└─ Gera dist/
└─ Tempo: ~2-3 minutos

Step 7: Setup Java 21
```yaml
- name: Setup Java 21
  uses: actions/setup-java@v3
  with:
    distribution: temurin
    java-version: 21
```
└─ Instala Java Development Kit v21
└─ Temurin distribution (OpenJDK)
└─ Tempo: ~2-3 minutos

Step 8: Add Android Platform
```yaml
- name: Add Android Platform
  run: npx cap add android
```
└─ Adiciona suporte Android ao Capacitor
└─ Cria estrutura android/
└─ Tempo: ~1-2 minutos

Step 9: Copy Web to Android
```yaml
- name: Copy Web to Android
  run: npx cap copy
```
└─ Copia dist/ para android/app/src/main/assets/public
└─ Sincroniza assets web com Android
└─ Tempo: ~30 segundos

Step 10: Assemble APK
```yaml
- name: Assemble APK
  run: |
    cd android
    ./gradlew assembleDebug
```
└─ Compila APK em modo debug
└─ Gradle Wrapper
└─ Tempo: ~5-8 minutos

Step 11: Upload APK
```yaml
- name: Upload APK
  uses: actions/upload-artifact@v4
  with:
    name: organistuda-debug-apk
    path: android/app/build/outputs/apk/debug/app-debug.apk
```
└─ Salva APK como artifact
└─ Disponível para download
└─ Tempo: ~1 minuto

4. COMO USAR O WORKFLOW
================================================================================

4.1 EXECUÇÃO AUTOMÁTICA
─────────────────────────────────────────────────────────────────────────────

Quando Executa:
- Toda vez que há push para main
- Automaticamente sem ação manual

Fluxo:
1. Desenvolvedor faz push para main
2. GitHub Actions dispara workflow automaticamente
3. APK é gerado
4. APK está disponível em Artifacts

4.2 EXECUÇÃO MANUAL
─────────────────────────────────────────────────────────────────────────────

Como Executar Manualmente:

1. Abrir GitHub: https://github.com/PietroLearnAndMake/OrganiStuda-

2. Ir para aba "Actions"
   └─ Clique em "Actions" no topo do repositório

3. Selecionar workflow "Build APK"
   └─ Lista de workflows à esquerda
   └─ Clique em "Build APK"

4. Clicar "Run workflow"
   └─ Botão no topo direito
   └─ Selecionar branch (main)
   └─ Clicar "Run workflow"

5. Aguardar conclusão
   └─ Status em tempo real
   └─ Verde = sucesso
   └─ Vermelho = erro

6. Fazer download do APK
   └─ Clicar na execução concluída
   └─ Aba "Artifacts"
   └─ Baixar "organistuda-debug-apk"

4.3 LOCALIZAÇÃO DO APK
─────────────────────────────────────────────────────────────────────────────

Após conclusão do workflow:

Artifacts:
- Nome: organistuda-debug-apk
- Arquivo: app-debug.apk
- Tamanho: ~30-50 MB
- Disponível por: 90 dias

Download:
1. GitHub Actions → Build APK → Execução concluída
2. Aba "Artifacts"
3. Clique em "organistuda-debug-apk"
4. Arquivo app-debug.apk será baixado

─────────────────────────────────────────────────────────────────────────────

Trigger:
- Push para main
- Execução manual (workflow_dispatch)

Jobs:

Tempo: ~15-20 minutos

Resultado:
- APK como artifact
- Sem GitHub Release

Uso:
- Build rápido
- Testes de desenvolvimento
- Builds sob demanda

6. CONFIGURAÇÕES FUTURAS
================================================================================

6.1 BUILD APK RELEASE (ASSINADO)
─────────────────────────────────────────────────────────────────────────────

Para gerar APK release (para Play Store):

Alterar step "Assemble APK":

```yaml
- name: Assemble APK
  run: |
    cd android
    ./gradlew assembleRelease
```

Ou com assinatura automática:

```yaml
- name: Assemble APK
  run: |
    cd android
    ./gradlew bundleRelease
```

Requer:
- Keystore configurado
- Variáveis de ambiente com credenciais
- Configuração de assinatura no build.gradle

6.2 SPLIT APK (POR ARQUITETURA)
─────────────────────────────────────────────────────────────────────────────

Para gerar APKs separados por arquitetura:

```yaml
- name: Assemble APK
  run: |
    cd android
    ./gradlew assembleDebug --parallel
```

Resultado:
- app-debug-arm64-v8a.apk
- app-debug-armeabi-v7a.apk
- app-debug-x86.apk
- app-debug-x86_64.apk

Benefício:
- Menor tamanho por dispositivo
- Melhor experiência de download

6.3 NOTIFICAÇÃO DE CONCLUSÃO
─────────────────────────────────────────────────────────────────────────────

Adicionar notificação ao workflow:

```yaml
- name: Notify on Success
  if: success()
  run: echo "APK build concluído com sucesso!"

- name: Notify on Failure
  if: failure()
  run: echo "APK build falhou!"
```

Ou com Slack/Discord:

```yaml
- name: Send Slack Notification
  uses: slackapi/slack-github-action@v1
  with:
    payload: |
      {
        "text": "APK build concluído!"
      }
```
