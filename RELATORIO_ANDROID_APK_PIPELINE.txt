3.1 SETUP JAVA
─────────────────────────────────────────────────────────────────────────────

Step Adicionado:

```yaml
- name: Setup Java
  uses: actions/setup-java@v3
  with:
    java-version: '17'
    distribution: 'temurin'
```

O que faz:
- Instala Java Development Kit (JDK) v17
- Temurin: Distribuição OpenJDK mantida pela Eclipse Foundation
- Necessário para compilar código Android com Gradle

Java 17:
- LTS (Long Term Support) até 2026
- Suportado por Gradle 8.x
- Compatível com Android SDK
- Melhor performance que Java 11

Temurin Distribution:
- Alternativa gratuita ao Oracle JDK
- Totalmente compatível com Android
- Mantida pela comunidade
- Recomendada para CI/CD

3.2 BUILD ANDROID APK
─────────────────────────────────────────────────────────────────────────────

Step Adicionado:

```yaml
- name: Build Android APK
  run: |
    npx cap add android
    npx cap copy
    cd android
    ./gradlew assembleDebug
```

Comandos Executados:

1. npx cap add android
   └─ Adiciona suporte Android ao Capacitor
   └─ Cria diretório android/
   └─ Configura projeto Android
   └─ Primeira execução: cria estrutura
   └─ Execuções posteriores: atualiza

2. npx cap copy
   └─ Copia assets web para Android
   └─ Copia dist/ → android/app/src/main/assets/public
   └─ Atualiza código nativo
   └─ Sincroniza configurações

3. cd android
   └─ Muda para diretório Android
   └─ Necessário para executar Gradle

4. ./gradlew assembleDebug
   └─ Compila APK em modo debug
   └─ Gradle Wrapper (versão específica)
   └─ Gera APK não assinado
   └─ Saída: android/app/build/outputs/apk/debug/app-debug.apk

Fluxo Completo:

```
Build Job:
├─ Checkout
├─ Setup Node.js
├─ Install pnpm
├─ Install dependencies
├─ Build application (pnpm build)
│  └─ Gera dist/ (PWA)
├─ Setup Java (v17)
│  └─ Instala JDK necessário
├─ Build Android APK
│  ├─ npx cap add android
│  ├─ npx cap copy
│  └─ ./gradlew assembleDebug
│     └─ Gera app-debug.apk
└─ Upload build artifacts
   ├─ dist/ (PWA)
   └─ android/app/build/outputs/apk/debug/app-debug.apk
```

3.3 UPLOAD DOS ARTIFACTS
─────────────────────────────────────────────────────────────────────────────

Antes:

```yaml
- name: Download build artifacts
  uses: actions/download-artifact@v4
  with:
    name: build-artifacts
    path: dist/

- name: Zip build
  run: zip -r dist.zip dist

- name: Upload Release Asset
  uses: actions/upload-release-asset@v1
  with:
    asset_path: ./dist.zip
    asset_name: organistuda-build-${{ github.run_number }}.zip
    asset_content_type: application/zip
```

Depois:

```yaml
- name: Download build artifacts
  uses: actions/download-artifact@v4
  with:
    name: build-artifacts
    path: ./

- name: Upload APK Release Asset
  uses: actions/upload-release-asset@v1
  with:
    asset_path: ./android/app/build/outputs/apk/debug/app-debug.apk
    asset_name: organistuda-v${{ github.run_number }}.apk
    asset_content_type: application/vnd.android.package-archive
```

Alterações:
1. Download path: dist/ → ./ (raiz)
   └─ Permite acessar artifacts em qualquer lugar

2. Removido: Zip build step
   └─ Não precisa mais de ZIP

3. Upload: dist.zip → app-debug.apk
   └─ Agora faz upload do APK

4. asset_name: organistuda-build-{n}.zip → organistuda-v{n}.apk
   └─ Nome mais apropriado para APK

5. asset_content_type: application/zip → application/vnd.android.package-archive
   └─ MIME type correto para APK

5. FLUXO DE EXECUÇÃO DO PIPELINE

   ├─ Install pnpm
   ├─ Install dependencies
   ├─ Run linter (pnpm lint)
   ├─ Run type check (pnpm check)
   └─ Run tests (pnpm test)
   
   Tempo: ~5-10 minutos

2. BUILD JOB (ubuntu-latest, needs: test)
   ├─ Checkout
   ├─ Setup Node.js 22.x
   ├─ Install pnpm
   ├─ Install dependencies
   ├─ Build application (pnpm build)
   │  └─ Gera dist/ (1.8 MB)
   ├─ Setup Java 17
   │  └─ Instala JDK (~500 MB)
   ├─ Build Android APK
   │  ├─ npx cap add android
   │  ├─ npx cap copy
   │  └─ ./gradlew assembleDebug
   │     └─ Compila APK (~30-50 MB)
   └─ Upload build artifacts
      ├─ dist/
      └─ app-debug.apk
   
   Tempo: ~15-25 minutos (primeira execução)
          ~10-15 minutos (execuções posteriores)

3. RELEASE JOB (ubuntu-latest, needs: build, if: push && main)
   ├─ Checkout
   ├─ Download build artifacts
   │  ├─ dist/
   │  └─ app-debug.apk
   ├─ Create Release
   │  └─ Tag: v{build_number}
   │  └─ Nome: Release v{build_number}
   └─ Upload APK Release Asset
      └─ organistuda-v{build_number}.apk
   
   Tempo: ~2-5 minutos

Total: ~25-40 minutos (primeira execução)
       ~15-25 minutos (execuções posteriores)
