└─ Faz checkout do repositório
└─ Tempo: ~10 segundos

Step 2: Setup Node.js
└─ Instala Node.js v20
└─ Tempo: ~30 segundos

Step 3: Setup PNPM
└─ Instala pnpm globalmente
└─ Tempo: ~30 segundos

Step 4: Install dependencies
└─ pnpm install
└─ Instala todas as dependências
└─ Tempo: ~2-3 minutos

Step 5: Build web assets
└─ pnpm build
└─ Compila React + TypeScript
└─ Gera dist/
└─ Tempo: ~2-3 minutos

Step 6: Setup Java 17
└─ Instala JDK 17
└─ Temurin distribution
└─ Necessário para Gradle
└─ Tempo: ~2-3 minutos

Step 7: Add Capacitor Android
└─ npx cap add android
└─ npx cap copy
└─ Cria estrutura Android
└─ Copia assets web
└─ Tempo: ~1-2 minutos

Step 8: Build APK Debug
└─ cd android && ./gradlew assembleDebug
└─ Compila APK em modo debug
└─ Saída: android/app/build/outputs/apk/debug/app-debug.apk
└─ Tamanho: ~30-50 MB
└─ Tempo: ~5-8 minutos

Step 9: Build APK Release
└─ cd android && ./gradlew assembleRelease
└─ Compila APK em modo release
└─ Saída: android/app/build/outputs/apk/release/app-release.apk
└─ Tamanho: ~15-30 MB (otimizado)
└─ Tempo: ~5-8 minutos
└─ ⚠️ Pode falhar sem keystore configurado

Step 10: Upload APKs
└─ actions/upload-artifact@v4
└─ Salva ambos APKs
└─ Nome: OrganiStuda-APK
└─ Tempo: ~1 minuto

4. APK DEBUG vs APK RELEASE
================================================================================

APK DEBUG
─────────────────────────────────────────────────────────────────────────────

Propósito:
- Testes e desenvolvimento
- Debugging
- Não assinado
- Não otimizado

Características:
- Tamanho: ~30-50 MB
- Assinatura: Debug key (gerada automaticamente)
- Otimização: Nenhuma
- ProGuard: Desativado
- Source maps: Inclusos

Localização:
android/app/build/outputs/apk/debug/app-debug.apk

Uso:
- Instalar em emulador
- Instalar em dispositivo físico
- Testes internos
- Debugging

Instalação:
```bash
adb install android/app/build/outputs/apk/debug/app-debug.apk
```

APK RELEASE
─────────────────────────────────────────────────────────────────────────────

Propósito:
- Produção
- Play Store
- Assinado
- Otimizado

Características:
- Tamanho: ~15-30 MB (otimizado)
- Assinatura: Release key (necessário configurar)
- Otimização: ProGuard/R8
- ProGuard: Ativado (obfuscação)
- Source maps: Removidos

Localização:
android/app/build/outputs/apk/release/app-release.apk

Uso:
- Publicar no Play Store
- Distribuição externa
- Produção

Publicação:
```bash
# No Play Store Console
# Upload do app-release.apk
```

COMPARAÇÃO
─────────────────────────────────────────────────────────────────────────────

| Aspecto | Debug | Release |
|---------|-------|---------|
| **Tamanho** | 30-50 MB | 15-30 MB |
| **Assinatura** | Debug key | Release key |
| **Otimização** | Nenhuma | ProGuard/R8 |
| **Obfuscação** | Não | Sim |
| **Source maps** | Sim | Não |
| **Tempo build** | 5-8 min | 5-8 min |
| **Uso** | Testes | Produção |
| **Play Store** | Não | Sim |

6. PRÓXIMAS OTIMIZAÇÕES
================================================================================

6.1 CONFIGURAR ASSINATURA AUTOMÁTICA
─────────────────────────────────────────────────────────────────────────────

Problema Atual:
- APK Release pode falhar sem keystore configurado
- Necessário configurar manualmente

Solução:
1. Gerar keystore
   ```bash
   keytool -genkey -v -keystore organistuda.keystore \
     -keyalg RSA -keysize 2048 -validity 10000 \
     -alias organistuda
   ```

2. Adicionar secrets no GitHub
   - KEYSTORE_FILE (base64 do keystore)
   - KEYSTORE_PASSWORD
   - KEY_ALIAS
   - KEY_PASSWORD

3. Configurar assinatura no workflow
   ```yaml
   - name: Sign APK
     run: |
       jarsigner -verbose -sigalg SHA256withRSA \
         -digestalg SHA-256 \
         -keystore organistuda.keystore \
         app-release.apk organistuda
   ```

6.2 PUBLICAR AUTOMATICAMENTE NO PLAY STORE
─────────────────────────────────────────────────────────────────────────────

Ferramenta: fastlane

Instalação:
```bash
gem install fastlane
```

Configuração:
```yaml
- name: Publish to Play Store
  run: |
    fastlane supply --apk android/app/build/outputs/apk/release/app-release.apk
```

6.3 SPLIT APK POR ARQUITETURA
─────────────────────────────────────────────────────────────────────────────

Alterar step "Build APK Release":

```yaml
- name: Build APK Release
  run: |
    cd android
    ./gradlew bundleRelease
```

Resultado:
- app-release.aab (Android App Bundle)
- Play Store gera APKs por arquitetura automaticamente
