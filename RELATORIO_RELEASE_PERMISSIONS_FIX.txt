================================================================================

O pipeline CI/CD estava falhando no step "actions/create-release@v1" com o erro:

❌ ERRO:
   Resource not accessible by integration

CONTEXTO:
   - Action: actions/create-release@v1
   - Causa: Falta de permissões no workflow
   - Permissão necessária: contents: write
   - Token: GITHUB_TOKEN (padrão)

IMPACTO:

   - Impossível publicar artefatos
   - Impossível criar GitHub Release
   - Impossível publicar no Play Store

CAUSA RAIZ:
   - GitHub Actions por padrão tem permissões mínimas
   - Criar release requer permissão explícita "contents: write"
   - Workflow não tinha declaração de permissões
   - GITHUB_TOKEN não tinha acesso para escrever conteúdo

3. DETALHES TÉCNICOS
================================================================================

GITHUB ACTIONS PERMISSIONS:

   O que são permissões?
   - Controle de acesso para GitHub Actions
   - Definem o que o workflow pode fazer
   - Aplicadas ao GITHUB_TOKEN automaticamente
   - Melhoram segurança e controle

   Permissões disponíveis:
   ┌──────────────────────────────────────────────────────────┐
   │ Permissão | Descrição                                    │
   ├──────────────────────────────────────────────────────────┤
   │ contents  | Ler/escrever conteúdo do repositório        │
   │ pull-requests | Gerenciar pull requests                 │
   │ issues    | Gerenciar issues                            │
   │ packages  | Publicar/deletar pacotes                    │
   │ statuses  | Criar/atualizar status de commit            │
   │ checks    | Criar/atualizar checks                      │
   └──────────────────────────────────────────────────────────┘

   Permissão "contents: write":
   - Permite criar releases
   - Permite fazer push de commits
   - Permite criar tags
   - Permite escrever em branches
   - Essencial para automação de release

ESTRUTURA DO WORKFLOW:

   Antes:
   ┌─────────────────────────────────────────┐
   │ name: Build & Test Pipeline             │
   │                                         │
   │ on:                                     │
   │   push:                                 │
   │     branches: [ main, develop ]         │
   │   pull_request:                         │
   │     branches: [ main, develop ]         │
   │                                         │
   │ jobs:                                   │
   │   test: ...                             │
   │   build: ...                            │
   │   release: ... ❌ SEM PERMISSÃO         │
   └─────────────────────────────────────────┘

   Depois:
   ┌─────────────────────────────────────────┐
   │ name: Build & Test Pipeline             │
   │                                         │
   │ permissions:                            │
   │   contents: write                       │
   │                                         │
   │ on:                                     │
   │   push:                                 │
   │     branches: [ main, develop ]         │
   │   pull_request:                         │
   │     branches: [ main, develop ]         │
   │                                         │
   │ jobs:                                   │
   │   test: ...                             │
   │   build: ...                            │
   │   release: ... ✅ COM PERMISSÃO         │
   └─────────────────────────────────────────┘

FLUXO DE EXECUÇÃO:

   1. Push para main/develop
      ↓
   2. GitHub Actions dispara workflow
      ↓
   3. Verifica permissões (permissions: contents: write)
      ↓
   4. Job: test (lint, check, test)
      ↓
   5. Job: build (build + upload artifacts)
      ↓
   6. Job: release (if: push && main)
      ├─ Download artifacts
      ├─ Create Release ✅ (agora tem permissão)
      └─ Upload Release Assets ✅ (agora tem permissão)
      ↓
   7. Release criado com sucesso

GITHUB_TOKEN:

   O que é:
   - Token automático gerado pelo GitHub
   - Disponível em cada workflow run
   - Permissões definidas por "permissions:"
   - Escopo: repositório atual

   Uso:
   - env: GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
   - Autenticação automática
   - Sem necessidade de configuração manual

   Segurança:
   - Token é específico para cada run
   - Expira após conclusão do workflow
   - Permissões limitadas por "permissions:"
   - Não pode acessar outros repositórios
