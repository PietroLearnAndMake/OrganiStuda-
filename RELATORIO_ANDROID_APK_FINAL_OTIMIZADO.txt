2. ARQUIVO FINAL
================================================================================

Localização: .github/workflows/android-apk.yml

Conteúdo:

```yaml
name: Build APK OrganiStuda

on:
  push:
    branches:
      - main
  workflow_dispatch:

jobs:
  build-apk:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repo
        uses: actions/checkout@v3

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: 20

      - name: Install pnpm
        run: npm install -g pnpm

      - name: Install dependencies
        run: pnpm install

      - name: Build web assets
        run: pnpm build

      - name: Setup Java 21
        uses: actions/setup-java@v3
        with:
          distribution: temurin
          java-version: 21

      - name: Install Capacitor Android platform
        run: |
          npx cap add android
          npx cap copy

      - name: Build APK Debug
        working-directory: android
        run: ./gradlew assembleDebug

      - name: Build APK Release
        working-directory: android
        run: ./gradlew assembleRelease

      - name: Upload Debug APK
        uses: actions/upload-artifact@v4
        with:
          name: organistuda-debug.apk
          path: android/app/build/outputs/apk/debug/app-debug.apk

      - name: Upload Release APK
        uses: actions/upload-artifact@v4
        with:
          name: organistuda-release.apk
          path: android/app/build/outputs/apk/release/app-release.apk
```

3.3 OTIMIZAÇÕES APLICADAS
─────────────────────────────────────────────────────────────────────────────

1. Java 21
   └─ Melhor compatibilidade com Gradle 8.x
   └─ Suporte para recursos mais recentes
   └─ LTS até 2026

2. working-directory
   └─ Mais limpo que cd android
   └─ Melhor legibilidade
   └─ Menos propenso a erros

3. upload-artifact@v4
   └─ Versão mais recente
   └─ Melhor performance
   └─ Suporte para múltiplos arquivos

4. Artifacts Separados
   └─ organistuda-debug.apk
   └─ organistuda-release.apk
   └─ Mais fácil de identificar
   └─ Melhor organização

5. Nomes Descritivos
   └─ Claro qual é debug e qual é release
   └─ Facilita download e testes
