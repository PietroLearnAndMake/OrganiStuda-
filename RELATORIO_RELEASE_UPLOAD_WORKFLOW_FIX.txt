1. PROBLEMA IDENTIFICADO
================================================================================

O pipeline CI/CD estava falhando no step de release com múltiplos problemas:

❌ PROBLEMAS:
   1. Step "Create Release" sem ID para referência
   2. Falta de step para comprimir o build (dist/)
   3. Upload tentava fazer upload de diretório em vez de arquivo ZIP
   4. asset_path apontava para ./dist (diretório) em vez de ./dist.zip (arquivo)

CONTEXTO:
   - Action: actions/create-release@v1
   - Action: actions/upload-release-asset@v1
   - Problema: Fluxo de release incompleto

IMPACTO:
   - Release não conseguia fazer upload do build
   - Artefatos não eram publicados
   - GitHub Release criada mas sem assets
   - Impossível publicar no Play Store

CAUSA RAIZ:
   - Create Release não tinha ID para outputs serem referenciados
   - Falta de step para criar arquivo ZIP
   - Upload tentava fazer upload de diretório (não suportado)
   - Workflow incompleto e desorganizado

================================================================================
2. SOLUÇÃO IMPLEMENTADA
================================================================================

✅ ALTERAÇÕES REALIZADAS:

   1. Adicionar ID ao step Create Release
      └─ id: create_release
   
   2. Adicionar step Zip build
      └─ run: zip -r dist.zip dist
   
   3. Corrigir asset_path no Upload Release Asset
      └─ ./dist → ./dist.zip

ARQUIVO MODIFICADO:
   .github/workflows/build.yml

LOCALIZAÇÃO DAS ALTERAÇÕES:
   - Linha 85: Adicionado id: create_release
   - Linhas 84-85: Adicionado step Zip build
   - Linha 108: Alterado asset_path para ./dist.zip

RESULTADO:
   ✅ Release job funciona completamente
   ✅ Build é comprimido em ZIP
   ✅ Arquivo ZIP é enviado para release
   ✅ GitHub Release tem assets para download

3. DETALHES TÉCNICOS
================================================================================

ALTERAÇÃO 1: ADICIONAR ID AO CREATE RELEASE

   Antes:
   ┌─────────────────────────────────────────┐
   │ - name: Create Release                  │
   │   uses: actions/create-release@v1       │
   │   env:                                  │
   │     GITHUB_TOKEN: ${{ ... }}            │
   │   with:                                 │
   │     tag_name: v${{ ... }}               │
   │     ...                                 │
   └─────────────────────────────────────────┘

   Depois:
   ┌─────────────────────────────────────────┐
   │ - name: Create Release                  │
   │   id: create_release                    │ ← NOVO
   │   uses: actions/create-release@v1       │
   │   env:                                  │
   │     GITHUB_TOKEN: ${{ ... }}            │
   │   with:                                 │
   │     tag_name: v${{ ... }}               │
   │     ...                                 │
   └─────────────────────────────────────────┘

   Impacto:
   - Step agora tem identificador único "create_release"
   - Outputs podem ser referenciados por: ${{ steps.create_release.outputs.upload_url }}
   - Essencial para o step de upload acessar a URL de upload

ALTERAÇÃO 2: ADICIONAR STEP ZIP BUILD

   Novo Step:
   ┌─────────────────────────────────────────┐
   │ - name: Zip build                       │
   │   run: zip -r dist.zip dist             │
   └─────────────────────────────────────────┘

   O que faz:
   - Comando: zip -r dist.zip dist
   - -r: Recursivo (inclui subdiretórios)
   - dist.zip: Nome do arquivo de saída
   - dist: Diretório a ser comprimido

   Resultado:
   - Cria arquivo dist.zip
   - Contém todo o conteúdo de dist/
   - Pronto para upload

   Posicionamento:
   - APÓS: Download build artifacts
   - ANTES: Create Release
   - Garante que ZIP está pronto antes de criar release

ALTERAÇÃO 3: CORRIGIR ASSET_PATH

   Antes:
   ┌─────────────────────────────────────────┐
   │ - name: Upload Release Assets           │
   │   uses: actions/upload-release-asset@v1 │
   │   env:                                  │
   │     GITHUB_TOKEN: ${{ ... }}            │
   │   with:                                 │
   │     upload_url: ${{ ... }}              │
   │     asset_path: ./dist                  │ ← DIRETÓRIO
   │     asset_name: ...                     │
   │     asset_content_type: application/zip │
   └─────────────────────────────────────────┘

   Depois:
   ┌─────────────────────────────────────────┐
   │ - name: Upload Release Asset            │
   │   uses: actions/upload-release-asset@v1 │
   │   env:                                  │
   │     GITHUB_TOKEN: ${{ ... }}            │
   │   with:                                 │
   │     upload_url: ${{ ... }}              │
   │     asset_path: ./dist.zip              │ ← ARQUIVO
   │     asset_name: ...                     │
   │     asset_content_type: application/zip │
   └─────────────────────────────────────────┘

   Impacto:
   - asset_path agora aponta para arquivo ZIP
   - Upload consegue fazer upload do arquivo
   - asset_content_type continua application/zip (correto)
